Health Journal Service

Profile UUID: 87ef07ff-4739-4527-b38f-b0e228de6ed3

Introduction

Health Journal records the health metrics produced by Angel Sensor. These are stored in non-volatile memory on the device in a structure similar to ring buffer (when full, the oldest records are overwritten to make space for the new ones). The entries of Health Journal can be accessed using the Health Journal Control Point.

In the scope of this service, Angel Sensor acts as the Server and the application using this service is the Client.

All characteristic values used with this service are transmitted with the least significant byte first (i.e. little endian).

Health Journal Entry characteristic

Characteristic UUID: 8b713a94-070a-4743-a695-fc58cb3f236b

The Health Journal Entry characteristic is used to send the recorded measurements from the Server to the Client. Included in the characteristic value are Timestamp and Flags fields, as well as one or more measurement fields such as Heart Rate, Temperature, and others.

When the Client Characteristic Configuration descriptor is configured for notifications the Health Journal Control Point can be used to control notifications of this characteristic.

Health Journal Control Point characteristic

Characterisitc UUID: 5ae61782-4a65-4202-a4da-db73406e38e8

For this service to operate, applications utilizing this service will need to ensure that the Client configures the Health Journal Control Point characteristic for indications (i.e., via the Client Characteristic Configuration descriptor).

The Client must perform a write to the Health Journal Control Point to execute a desired procedure at the Server.

Write format

Each write to the Health Journal Control Point must contain the following fields:

Field name Size Data format
Op code 8bit See Op-Codes
Operator 8bit See Operators-Operands below
Operand Variable See Operators-Operands below

Op-Codes:

Op Code Operation
1 Query entries
2 Delete entries
3 Abort operation
4 Query number of entries
0, 5-255 Reserved, must not be used

Operators/Operands:

Operator Code Operand
0 All entries No operands used
1 First entry No operands used
2 Last entry No operands used
3 Less than or equal to <max timestamp>
4 Greater than or equal to <min timestamp>
5 Within range <min timestamp, max timestamp>
6-255 Reserved, must not be used  

Response/error codes

Response codes of 0x80 and higher indicate an erroneous operation result.

Code Info
0 Success
0x80 Op code not supported
0x81 Operator not supported
0x82 Operand not supported
0x83 Client Characteristic Configuration Descriptor improperly configured, either notifications or indications must be on
0x84 Invalid operator format
0x85 Invalid operand format
0x86 Abort unsuccessful, sent if request for Abort cannot be completed
0x87 Procedure not completed, sent if unable to complete a procedure for any reason
0x88 Procedure already in progress
0x89 Not found, no results meet the last query criteria
0x9F Generic/unspecified error

Protocol Revision Characteristic

Characteristic UUID: 3b8e7983-133a-4a0f-90fc-82006ed55505

This characteristic is a constant value. It defines the revision number of the current service specification. Applications using this service will need to read the revision value in order to establish the particular service protocol supported by the Client.

Field name Format Info
Protocol Revision uint8 0-255
